# 下阶段增强任务

> 这份文档用于内部查阅，记录提取脚本和 skill 工具链的下一阶段增强方向。

## 已完成到什么程度

当前版本已经完成：

- 基础视觉 token 提取
- 组件样本提取
- 组件变体初步聚类
- 基础布局证据提取
- 基础响应式证据提取
- bundle 标准化、校验、拆分
- skill 与工具链整合

但它还没有到“强工程级提取器”的程度。

## 下一阶段重点

## 1. 多 viewport 自动采样

目标：

- 在 desktop / tablet / mobile 多宽度下自动采集
- 自动合并 viewport 差异
- 直接输出更可信的 responsive 证据

原因：

- 现在的响应式证据仍以“当前 viewport + media query”组合为主
- 这会让移动端结构信息不够强

## 2. 真实状态采样

目标：

- 对代表性按钮、输入框、导航项采样 hover / focus / active / disabled 的最终 computed styles

原因：

- 现在更多是从 stylesheet rule 提取状态规则
- 还不是“真实组件在该状态下最终长什么样”

## 3. 更强的页面骨架识别

目标：

- 识别 hero
- 识别 feature grid
- 识别 pricing
- 识别 testimonial
- 识别 FAQ
- 识别 dashboard overview
- 识别 docs article

原因：

- 现在 layout 证据已经有 container / grid / section
- 但还没有足够强的 page pattern 识别能力

## 4. 更强的 section rhythm 提取

目标：

- 更准确地提取 section gap
- 识别标题到正文间距
- 识别正文到 CTA 间距
- 识别卡片组内部节奏

原因：

- 这部分直接影响 AI 最终生成页面时是否“舒展”

## 5. 伪元素与装饰系统增强

目标：

- 更系统地捕捉 `::before` / `::after`
- 增强 gradient、mask、clip-path、mix-blend-mode、background-size、background-position 等装饰信息

原因：

- 很多品牌感和高级感恰恰来自装饰层，而不是主组件本体

## 6. 组件语义分类增强

目标：

- 更准确地区分 primary / secondary / ghost / disabled
- 更准确地区分 filled card / flat card / outline card
- 更准确地区分 top nav / sidebar / tab nav

原因：

- 当前聚类已经开始做，但还主要基于样式签名
- 后面可以引入更强的语义规则

## 7. 低 token 输出继续压缩

目标：

- 继续优化 summary
- 进一步控制长页面输出体积
- 把高价值字段前置，把低价值字段延后

原因：

- 这个项目的核心卖点之一就是节省 token

## 8. 生成层进一步升级

目标：

- 从 normalized bundle 更稳定地生成 `DESIGN.md`
- 新增从 layout evidence 生成 `LAYOUT.md`
- 减少生成阶段自由发挥

原因：

- 提取层越强，生成层越应该少“猜”

## 建议优先级

建议优先顺序：

1. 多 viewport 自动采样
2. 真实状态采样
3. 页面骨架识别
4. section rhythm 细化
5. 组件语义分类增强
6. 装饰系统增强
7. 输出压缩优化
8. 生成层进一步升级
