# Roadmap

## Phase 1. Foundation

Goal:

- establish the project architecture
- define the skill workflow
- define the target extraction schema

Status:

- in progress

Deliverables:

- project README
- upstream analysis
- design reference
- layout reference
- extraction bundle schema

## Phase 2. Extraction Upgrade

Goal:

- evolve the current prototype script into a fuller design intelligence collector

Priority upgrades:

- layout container extraction
- section rhythm extraction
- component variant clustering
- computed state capture for representative elements
- responsive snapshots across viewport sizes
- decorative effect extraction

## Phase 3. Dual-Document Generation

Goal:

- generate both `DESIGN.md` and `LAYOUT.md` from one normalized bundle

Deliverables:

- stable prompt contract
- output templates
- low-token synthesis workflow

## Phase 4. Reusable Packaging

Goal:

- turn extracted systems into reusable style skills

Deliverables:

- output package structure
- reusable `SKILL.md` template
- example extracted bundles

## Phase 5. Quality Evaluation

Goal:

- test whether extracted systems actually improve generated frontend quality

Evaluation targets:

- desktop spacing quality
- mobile adaptation quality
- section rhythm quality
- structural similarity to target sites
- reduction in model drift between runs
